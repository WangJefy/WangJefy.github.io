单向数据流(One-Way Data Flow)

  1. 所有的 prop 都 使得其父子prop之间形成了一个 单向向下绑定(one-way-down binding)：父级prop的更新会向下流动到子组件中，但是反过来不行！
  2. 这样为了防止 从子组件意外改变父组件的状态，从而导致你的应用的数据流向难以理解。
  3. 此外，每次父级组件发生更新时，子组件中所有的prop都将会刷新为最新的值。
  4. 这意味着 你 不应该 在一个子组件内部改变prop，如果你这样做了，Vue会在浏览器的控制台中发出警告。

非Prop的特性(Non-Prop Attributes)

  1. 一个非 prop 特性是指传向一个组件，但是该组件并没有相应 prop 定义的特性。
  2. A non-prop attribute is an attribute that is passed to a component, but does not have a corresponding prop defined.
  3. 非 prop 的特性 往往 可以 直接传入组件 ，而 不需要定义 相应的 prop
  4. 尽管为组件定义明确的 prop 是推荐的传参方式，但组件的作者却并不总能预见到组件被使用的场景。
  5. 所以，组件可以接收任意传入的特性，这些特性都会被添加到组件的根元素上
